# Agent instructions

- Each contribution/MR must have a changelog entry.
- Use `.agents/skills/release-plz-changelog/SKILL.md` when making, reviewing, committing, or preparing PR/MR text.
- Prefer a release-plz-compatible Conventional Commit summary as the changelog entry, for example `fix: ...`, `feat: ...`, `docs: ...`, `ci: ...`, or `chore: ...`.
- Let release-plz update `CHANGELOG.md` and versions through its release PR; do not manually bump versions for ordinary changes.
